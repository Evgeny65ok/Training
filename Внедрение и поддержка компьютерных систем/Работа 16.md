## Теоретическая часть

**Групповые политики** — это инструмент администратора, позволяющий управлять параметрами системы и пользователей (от интерфейса до ограничений безопасности). В Windows 10 редактор *локальной* групповой политики (gpedit.msc) доступен в версиях Pro и Enterprise.

Окно редактора разделено на две основные категории:

1.  **Конфигурация компьютера** — политики действуют на всех пользователей и применяются до входа в систему.
2.  **Конфигурация пользователя** — политики действуют только на текущего пользователя после его входа.

Большинство интересных политик (не связанных с безопасностью) находятся в ветви **«Административные шаблоны»**.

### 1. Запуск редактора локальной групповой политики

Операционная система Windows 10 установлена на . Вход в систему выполнен с использованием учетной записи, обладающей правами администратора.

Для запуска редактора локальной групповой политики выполнил следующие действия:

1.  Нажал сочетание клавиш **Win + R** (открытие окна «Выполнить»).
2.  В поле «Открыть» ввел команду **`gpedit.msc`**.

### 2. Навигация и фильтрация политик (поиск)

Учитывая, что в Windows 10 существует около 3000 политик, разбросанных по десяткам папок, поиск нужной политики вручную затруднителен. Я использовал механизм **фильтрации** для поиска конкретной политики.

1.  В левой панели перешел по пути: **Конфигурация пользователя → Административные шаблоны → Все параметры**.
2.  В меню **«Действие»** выбрал пункт **«Параметры фильтра...»**.
3.  В открывшемся диалоговом окне установил флажок **«Включить фильтры по ключевым словам»**.
4.  В текстовое поле «Фильтры по словам» ввел слово **«мест»** (так как искал политику «Элементы, отображаемые в панели мест»).
5.  Убедился, что установлен флажок поиска по **«Заголовку параметра политики»**.
6.  Раскрывающийся список уровня соответствия оставил в значении **«Любой»** (по умолчанию).



7.  Нажал кнопку **OK**.
8.  На панели инструментов редактора активировал кнопку **«Фильтр»** (или выбрал меню Действие → Фильтр).
9.  В правой панели отобразились только те политики, которые соответствуют критерию фильтрации.

### 3. Настройка политики (Удаление кнопок из окна безопасности Windows)

В качестве примера настройки я выбрал политики, отвечающие за отображение кнопок в окне безопасности Windows (вызываемом нажатием Ctrl+Alt+Delete).

1.  В окне редактора локальной групповой политики последовательно развернул ветви: **Конфигурация пользователя → Административные шаблоны → Система → Варианты действий после нажатия CTRL+ALT+DEL**.
2.  Дважды щелкнул на политике **«Запретить блокировку компьютера»**.
3.  В открывшемся диалоговом окне политики щелкнул на переключателе **«Включить»**, а затем нажал кнопку **OK**.

 


4.  Аналогичным образом включил политику **«Удалить диспетчер задач»**, чтобы отключить соответствующую кнопку.

### 4. Проверка применения групповых политик

Чтобы изменения вступили в силу без перезагрузки системы, выполнил принудительное обновление групповых политик.

1.  Запустил командную строку от имени администратора (Пуск → правой кнопкой мыши на «Командная строка» → Запуск от имени администратора).
2.  Ввел команду **`gpupdate /force`** и нажал Enter.
3.  Дождался сообщения об успешном обновлении политик.
4.  Нажал сочетание клавиш **Ctrl + Alt + Delete**.
5.  В открывшемся окне безопасности Windows кнопки **«Блокировать компьютер»** и **«Запустить диспетчер задач»** стали неактивны (отсутствуют).

### 5. Дополнительная настройка (Увеличение списка недавних документов)

Для закрепления материала выполнил настройку политики, увеличивающей количество отображаемых недавних документов в меню «Пуск».

1.  В редакторе локальной групповой политики перешел по пути: **Конфигурация пользователя → Административные шаблоны → Компоненты Windows → Проводник Windows**.
2.  В правой панели дважды щелкнул на политике **«Максимальная длина списка "Недавние документы"»**.
3.  В открывшемся окне щелкнул на переключателе **«Включить»**.

5.  Нажал кнопку **OK**.

    

6.  Для применения изменений снова выполнил команду `gpupdate /force`.

## Контрольные вопросы

**1. Что такое групповые политики?**  
Групповые политики — это набор правил и настроек, которые управляют рабочей средой пользователей и конфигурацией компьютера в ОС Windows. Они позволяют централизованно (или локально) ограничивать возможности пользователей, настраивать интерфейс, параметры безопасности, реестр и устанавливать различные ограничения.

**2. Категории объектов групповых политик?**  
В редакторе локальных групповых политик существуют две основные категории:
- **Конфигурация компьютера** — политики, которые применяются ко всем пользователям данного компьютера и действуют до входа пользователя в систему.
- **Конфигурация пользователя** — политики, которые применяются только к текущему пользователю после его входа в систему.

**3. Разделы групповых политик**  
Основные разделы внутри категорий:
- **Конфигурация программ** — управление установкой и работой приложений.
- **Конфигурация Windows** — настройка безопасности, сценариев входа/выхода, папок.
- **Административные шаблоны** — основной раздел для тонкой настройки интерфейса, компонентов ОС (Проводник, меню Пуск, Панель задач и др.).

**4. Как запустить редактор групповых политик?**  
Существует несколько способов запуска:
- Нажать сочетание клавиш **Win + R**, ввести команду **`gpedit.msc`** и нажать Enter.
- Нажать кнопку **Пуск**, в строке поиска ввести `gpedit.msc` и выбрать соответствующее приложение.
- Открыть **Командную строку** или **PowerShell**, ввести команду `gpedit.msc` и нажать Enter.

**5. Для чего нужна фильтрация политик в редакторе локальной групповой политики?**  
Фильтрация политик критически необходима, так как в современных версиях Windows насчитывается несколько тысяч доступных политик (более 3000). Без фильтрации по ключевым словам или состоянию политики найти нужный параметр практически невозможно. Фильтрация позволяет усечь «безбрежный ландшафт» доступных политик до приемлемого размера, отображая только те пункты, которые соответствуют заданным критериям поиска.

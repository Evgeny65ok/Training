# Краткие теоретические сведения: Файловая структура OS Linux## 1. Концепция файловой системы
Основной принцип OS Linux — «Всё есть файл». Файловая система имеет единую иерархическую структуру, напоминающую перевернутое дерево. Конечными узлами дерева являются файлы, а промежуточными — каталоги (папки).

* Корневой каталог (/) — вершина всей файловой системы.
* Путь к файлу — цепочка каталогов, разделенных косой чертой / (например, /home/user/file.txt).

## 2. Основные каталоги Linux и их назначение

* /bin — основные системные утилиты и команды (доступны всем).
* /sbin — системные утилиты для администрирования (доступны superuser).
* /boot — файлы для загрузки операционной системы и ядро Linux.
* /dev — файлы устройств (диски, терминалы, мышь).
* /etc — конфигурационные файлы системы и программ.
* /home — домашние каталоги пользователей (личные файлы).
* /root — личный домашний каталог суперпользователя Root.
* /lib / /lib64 — системные библиотеки для работы программ.
* /media / /mnt — точки монтирования съемных дисков и флешек.
* /proc — виртуальная ФС с информацией о процессах и ядре.
* /sys — информация об аппаратной части компьютера.
* /tmp — временные файлы (очищаются при перезагрузке).
* /usr — пользовательские программы, документация и библиотеки.
* /var — изменяемые данные: лог-файлы, базы данных, почта.

       [ / ] (Корень)
        ├── bin/
        ├── boot/
        ├── dev/
        ├── etc/
        ├── home/
        │    └── user/
        ├── root/
        └── var/

## 3. Типы файлов в Linux
При просмотре списка файлов (команда ls -l) первый символ строки определяет его тип:

* - — Обычный файл (текст, исполняемая программа, картинка).
* d — Каталог (директория, содержащая другие файлы).
* l — Символическая ссылка (ярлык на другой файл).
* c / b — Файлы устройств (символьные или блочные).

## 4. Права доступа и безопасность
Каждый объект в Linux имеет владельца (User), группу (Group) и права для остальных пользователей (Others).
## Три типа прав:

* r (Read) — чтение.
* w (Write) — запись/изменение.
* x (Execute) — выполнение файла / вход в каталог.

## Формат отображения прав (ls -l):
Строка прав состоит из 9 символов, разбитых на три группы:
rwx r-x r--

   1. rwx — права владельца (может читать, писать, запускать).
   2. r-x — права группы (может читать и запускать).
   3. r-- — права всех остальных (только чтение).

## Управление правами:

* Изменение прав: chmod [права] [файл] (например, chmod 755 script.sh).
* Изменение владельца: chown [пользователь] [файл].

## 1. Файлы в Linux
В Linux файлом считается всё. Это обычные данные, каталоги, устройства, порты и процессы.
## 2. Связи с файлами (Ссылки)
Ссылки создают альтернативные имена для файлов.

* Жесткие (Hard links): указывают на тот же индексный дескриптор (inode). Удаление оригинала не уничтожает данные. Создание: ln файл ссылка.
* Символические (Symlinks): ярлыки, указывающие на путь к файлу. Ломаются при удалении оригинала. Создание: ln -s файл ссылка.

## 3. Атрибуты файлов
Атрибуты определяют тип файла, права доступа (чтение, запись, выполнение), владельца, группу и время изменения.

* Просмотр: команда ls -l.
* Изменение прав: команда chmod.
* Изменение владельца: команда chown.

## 4. Создание и удаление

* Файлы: создание — touch, echo >, nano; удаление — rm.
* Каталоги: создание — mkdir; удаление — rmdir (пустые) или rm -r (с содержимым).

## 5. Поиск по шаблону
Это поиск файлов с использованием спецсимволов (wildcards):

* * — любая строка любой длины.
* ? — один любой символ.
* [...] — любой символ из указанного диапазона.

## 6. Список пользователей в файл
Команда: who > users.txt (или w > users.txt).
